# Dialog

Bundle name: `"components/dialog"`

Implementation: [dialog.css](./dialog.css)

## Classes

### `dialog`

Applies to the components: `ch-dialog`

A dialog component is an overlay element designed to capture user attention for tasks like confirmations, prompts, or additional information. It appears on top of the main content with a backdrop, requiring user interaction to proceed

### `dialog-footer`

Applies to the components: Footer container within `ch-dialog`

Base class for dialog footer sections. Provides padding and layout structure for action buttons. By default, content is aligned to the end (right)

### `dialog-footer-center`

Applies to the components: Footer container within `ch-dialog`

Variant of dialog-footer that centers the footer content horizontally

### `dialog-footer-space-between`

Applies to the components: Footer container within `ch-dialog`

Variant of dialog-footer that distributes footer content with space-between alignment, typically used when you have actions on both left and right sides

### `dialog-footer-start`

Applies to the components: Footer container within `ch-dialog`

Variant of dialog-footer that aligns footer content to the start (left)
